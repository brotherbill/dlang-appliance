# Mission 2 — D Learning Appliance Production Doctrine  
# Deterministic. Parent‑Grade. Operator‑Grade. Assembly‑Line Safe.

Mission 2 defines the complete, reproducible, assembly‑line process for producing a locked‑down D Learning Appliance for parents, students, and institutions.  
This mission governs how appliances are cloned, customized, secured, shipped, supported, and eventually traded in or refurbished.

A man’s got to know his limitations.  
This mission defines the appliance’s.

---

## 1. PURPOSE OF THE APPLIANCE
The D Learning Appliance is a **plug‑and‑play**, **parent‑grade**, **operator‑grade** mini computer that delivers:

- the D language learning environment  
- the RCA (Root Cause Archive) companion app  
- the curriculum pipeline  
- the lockdown and safety model  
- the backup and restore workflow  
- the subscription and warranty model  

The appliance must be predictable, safe, tamper‑resistant, and supportable at scale.

---

## 2. WHO THE APPLIANCE SERVES
The appliance is designed for:

- parents teaching their children  
- students working independently  
- tutoring centers  
- homeschool co‑ops  
- after‑school programs  
- libraries  
- prisons and rehab centers  
- any environment requiring locked‑down, reproducible learning hardware  

The appliance must work for both high‑trust and low‑trust environments.

---

## 3. THE GOLDEN MASTER
All appliances originate from a **single Golden Master** image that contains:

- OS baseline  
- D toolchain  
- VS Code baseline  
- curriculum scaffolding  
- RCA baseline  
- lockdown Level 1 defaults  
- parent‑grade settings  
- operator‑grade logs  

The Golden Master is never modified directly.  
All changes flow through doctrine, then through controlled rebuilds.

---

## 4. PARENT INPUTS (THE THREE FIELDS)
Parents provide exactly three fields:

1. **Parent Name**  
2. **Child Name**  
3. **Restricted Username** (for the student account)

No additional inputs are accepted.  
No optional fields.  
No hidden fields.  
No drift.

These three fields drive the customization step.

---

## 5. THE ASSEMBLY LINE
Mission 2 is an **assembly‑line process**:

1. Start with the Golden Master.  
2. Clone it to a new device.  
3. Apply the three parent fields.  
4. Apply lockdown level.  
5. Apply RCA binding.  
6. Apply hardware checks (including Kensington lock requirement).  
7. Run operator‑grade verification.  
8. Package and ship.

No parallel steps.  
No improvisation.  
No undocumented actions.

---

## 6. LOCKDOWN LEVELS
The appliance supports three lockdown levels:

### **Level 1 — Parent‑Grade**
- Safe defaults  
- No destructive access  
- Curriculum‑only environment  
- RCA enabled  
- Updates controlled  

### **Level 2 — Operator‑Grade**
- Stronger restrictions  
- No package installs  
- No system settings access  
- No browser drift  
- RCA mandatory  

### **Level 3 — Institutional**
- Maximum lockdown  
- No removable storage  
- No external accounts  
- No escape paths  
- Physical security required (Kensington lock mandatory)  
- Irreversible without operator intervention  

Lockdown levels must be documented, reproducible, and testable.

---

## 7. PHYSICAL SECURITY REQUIREMENTS
Every appliance must meet physical security standards:

- Kensington lock slot required  
- Steel‑braided cable lock required  
- Fixed anchor point required  
- Cable routing with no slack  
- Device must not “take legs”  

If physical security cannot be guaranteed, the device cannot ship.

---

## 8. RCA (ROOT CAUSE ARCHIVE) BINDING
Each appliance must bind to RCA:

- logs  
- configs  
- backups  
- restore points  
- lockdown state  

RCA ensures supportability, warranty validation, and parent trust.

No appliance ships without RCA binding.

---

## 9. WARRANTY, SUBSCRIPTION, AND SUPPORT
Mission 2 governs:

- subscription activation  
- warranty validation  
- trade‑in pipeline  
- refurb pipeline  
- support escalation  
- replacement rules  

All support actions must be deterministic and documented.

---

## 10. TRADE‑IN AND REFURB PIPELINE
Returned appliances enter the refurb pipeline:

1. Intake  
2. RCA wipe  
3. Hardware inspection  
4. Lockdown reset  
5. Golden Master re‑clone  
6. Reassignment or recycling  

No appliance is reused without full RCA wipe and re‑clone.

---

Mission 2 must remain consistent across:

- dlang-appliance  
- dlang-school  
- from-dead-box-to-dev-box  

Terminology, workflow, and doctrine must match exactly.
