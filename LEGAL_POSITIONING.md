# Legal Positioning for the D Learning Appliance  
# Clear Boundaries. Clear Obligations. Zero Ambiguity.

This file defines the legal posture of the D Learning Appliance.  
It explains what is warranted, what is not, what is supported, what is not, and why the model avoids liability while preserving user freedom.

A man’s got to know his limitations.  
This file defines the appliance’s.

---

## 1. USER HARDWARE OWNERSHIP
The user owns the physical device.

They may:

- install any OS  
- wipe the drive  
- dual‑boot  
- replace hardware  
- install any software  
- modify the filesystem  
- use the device for any purpose  

The hardware is theirs.  
Their rights are unrestricted.

This eliminates “right‑to‑repair” and “device‑ownership” legal exposure.

---

## 2. APPLIANCE AS A MANAGED ENVIRONMENT
The D Learning Appliance is a **software environment**, not the hardware.

It is:

- a controlled curriculum platform  
- a deterministic Golden Master image  
- an RCA‑bound system  
- a lockdown‑governed environment  
- a reproducible, auditable subsystem  

The appliance is **separate** from the general‑purpose computer.

This separation eliminates liability for user actions outside the appliance.

---

## 3. WARRANTY SCOPE
The warranty applies to:

- the appliance  
- the Golden Master  
- the curriculum pipeline  
- the RCA system  
- the lockdown system  
- operator‑grade assembly defects  

The warranty does **not** apply to:

- user‑installed software  
- user‑modified OS  
- user‑modified filesystem  
- external accounts  
- drift  
- tampering  
- wiping the appliance  
- replacing the appliance with another OS  

This eliminates liability for user‑caused failures.

---

## 4. WARRANTY OBLIGATIONS
The warranty guarantees:

- correction of internal defects  
- operator‑controlled prioritization  
- safe, deterministic fixes  
- no compromise to RCA, lockdown, or Golden Master  

The warranty does **not** guarantee:

- turnaround time  
- speed  
- SLAs  
- customer‑dictated priorities  
- customer‑dictated timelines  

This eliminates the #1 cause of software‑related lawsuits:  
**implied promises of speed or performance.**

---

## 5. DEFECT PRIORITIZATION
Internal defects are corrected according to operator‑defined priority.

- cosmetic defects → lowest priority  
- functional defects → highest priority  

This eliminates liability for “you fixed X before Y” claims.

---

## 6. USER FREEDOM OUTSIDE THE APPLIANCE
The user may:

- exit the appliance  
- use the computer normally  
- install any software  
- browse the web  
- use external accounts  
- run VMs  
- do anything they want  

Their freedom is unrestricted **outside** the appliance.

This eliminates liability for user‑initiated system changes.

---

## 7. APPLIANCE INTEGRITY REQUIREMENT
The appliance must remain:

- deterministic  
- reproducible  
- auditable  
- lockdown‑consistent  
- RCA‑consistent  
- Golden‑Master‑consistent  

If the user breaks appliance integrity, the warranty is void.

This eliminates liability for tampering.

---

## 8. NO IMPLIED PROMISES
The doctrine explicitly avoids:

- implied performance guarantees  
- implied uptime guarantees  
- implied support guarantees  
- implied timelines  
- implied SLAs  

This eliminates the #2 cause of software‑related lawsuits:  
**ambiguity.**

---

## 9. INDUSTRY ALIGNMENT
This model aligns with:

- enterprise MDM laptops  
- school‑issued Chromebooks  
- iPads with managed profiles  
- Steam Deck (Gaming Mode vs Desktop Mode)  
- Android work profiles  

The hardware is free.  
The managed environment is controlled.

This is a legally conservative, widely accepted model.

---

## 10. LIABILITY MINIMIZATION SUMMARY
The doctrine avoids liability by:

- separating hardware ownership from appliance warranty  
- defining explicit boundaries  
- defining explicit exclusions  
- defining explicit defect priorities  
- defining no turnaround times  
- defining no SLAs  
- defining no customer‑dictated obligations  
- defining deterministic rules for support  
- defining deterministic rules for warranty  
- defining deterministic rules for integrity  

Ambiguity is eliminated.  
Implied promises are eliminated.  
Customer‑dictated expectations are eliminated.

This is the safest possible legal posture.

- dlang-appliance

- from-dead-box-to-dev-box  

If one changes, all must change.
